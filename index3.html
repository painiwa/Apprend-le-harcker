Leçon 3 : Réseaux et Protocoles

Introduction

Comprendre les réseaux et les protocoles est crucial pour tout hacker éthique. Cela permet de savoir comment les données circulent entre les appareils et comment détecter ou prévenir des vulnérabilités.

Concepts essentiels

  • IP : Chaque appareil sur un réseau a une adresse IP unique pour s’identifier et communiquer.
  • TCP (Transmission Control Protocol) : Assure la transmission fiable des données entre deux machines.
  • UDP (User Datagram Protocol) : Transmission rapide des données sans vérification d’erreur, utile pour le streaming ou les jeux.
  • Ports : Points de communication sur un appareil. Chaque service (HTTP, FTP, SSH) utilise un port spécifique.

Commandes de base sous Linux


# Vérifier la configuration réseau
ifconfig

# Afficher l'adresse IP actuelle et le masque de sous-réseau
ip addr show

# Tester la connexion à un serveur (ping)
ping google.com

# Vérifier les ports ouverts sur l'ordinateur local (netstat)
netstat -tuln

Explications pratiques

  • ifconfig : affiche les interfaces réseau et leurs paramètres (IP, masque, état).
  • ip addr show : alternative moderne à ifconfig pour voir les adresses IP.
  • ping google.com : teste la connectivité entre votre machine et un serveur distant.
  • netstat -tuln : liste les ports TCP/UDP ouverts et les services qui les utilisent.

Conseils pour débutants

  • Pratiquez ces commandes dans un terminal Linux ou une machine virtuelle.
  • Notez les adresses IP et les ports pour comprendre la structure du réseau.
  • Combinez ping et netstat pour identifier rapidement les problèmes ou services actifs.
  • Utilisez ces connaissances pour mieux comprendre les scans et tests de sécurité futurs.

Résumé

Cette leçon a présenté les bases des réseaux et des protocoles essentiels pour la sécurité. Vous devez maintenant être capable de :

  • Identifier l’adresse IP d’une machine.
  • Comprendre la différence entre TCP et UDP.
  • Vérifier la connectivité et les ports ouverts sur un système.
