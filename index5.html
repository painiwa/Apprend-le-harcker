Leçon 5 : Test de pénétration

Introduction

Le test de pénétration (ou pentest) consiste à simuler des attaques sur un système pour détecter ses vulnérabilités avant qu’un hacker malveillant ne les exploite. C’est une étape essentielle pour la sécurité informatique.

Étapes principales d’un test de pénétration

  • Reconnaissance : Collecter des informations sur la cible (IP, domaine, services).
  • Scan de vulnérabilités : Identifier les ports ouverts, services actifs et vulnérabilités connues.
  • Exploitation contrôlée : Tester les failles trouvées dans un environnement sûr.
  • Rapport : Documenter les découvertes et proposer des solutions correctives.

Exemple pratique : Scanner un hôte avec Nmap


# Scanner les ports et services d'un hôte
nmap -sV 192.168.1.1

# Explication :
# -sV : détecte la version des services
# 192.168.1.1 : adresse IP de la cible

Explications détaillées

  • nmap : outil puissant pour découvrir les ports ouverts et services actifs.
  • -sV : permet de détecter la version des services pour identifier les vulnérabilités potentielles.
  • Les scans doivent toujours être effectués avec l’autorisation explicite de la cible.

Conseils pour débutants

  • Pratiquez d’abord sur votre propre réseau ou sur des machines virtuelles.
  • Documentez toutes vos étapes pour créer un rapport clair.
  • Ne tentez jamais de scanner des systèmes sans autorisation, cela est illégal.
  • Combinez Nmap avec d’autres outils comme Nikto ou OpenVAS pour des tests plus complets.

Résumé

Cette leçon a présenté les bases du test de pénétration et comment utiliser Nmap pour scanner des hôtes. Vous devez maintenant être capable de :

  • Comprendre les étapes d’un pentest.
  • Scanner un hôte pour identifier ports et services.
  • Appliquer ces connaissances dans un environnement sécurisé et légal.
