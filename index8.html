Leçon 8 : Cryptographie et Hashing

Introduction

La cryptographie est utilisée pour sécuriser les informations sensibles et assurer leur confidentialité et intégrité. Le hashing permet de transformer des données en une valeur unique et fixe, souvent utilisée pour stocker les mots de passe en toute sécurité.

Principaux concepts

  • Hash : Valeur unique générée à partir de données. Exemple : SHA-256, MD5.
  • Chiffrement symétrique : Une seule clé est utilisée pour chiffrer et déchiffrer.
  • Chiffrement asymétrique : Utilise une paire de clés (publique et privée) pour sécuriser les échanges.

Exemple pratique : Hasher un mot de passe en JavaScript


async function hashPwd(pwd) {
  // Convertit la chaîne de caractères en tableau d'octets
  const enc = new TextEncoder();

  // Calcule le hash SHA-256
  const hash = await crypto.subtle.digest('SHA-256', enc.encode(pwd));

  // Transforme le hash en chaîne hexadécimale
  return Array.from(new Uint8Array(hash))
              .map(b => b.toString(16).padStart(2, '0'))
              .join('');
}

// Exemple d'utilisation
hashPwd('monMotDePasse123').then(console.log);

Explications détaillées

  • TextEncoder() : convertit la chaîne en tableau d’octets pour le traitement.
  • crypto.subtle.digest('SHA-256', ...) : calcule le hash SHA-256 des données.
  • Uint8Array : tableau de bytes représentant le hash.
  • La conversion en hexadécimal permet de stocker le hash sous forme lisible et utilisable.

Conseils pour débutants

  • Ne jamais stocker de mots de passe en clair.
  • Utiliser des algorithmes de hash sécurisés comme SHA-256 ou bcrypt pour les mots de passe.
  • Pour le chiffrement, privilégier AES (symétrique) ou RSA/ECC (asymétrique) selon le besoin.
  • Pratiquez sur de petits scripts pour comprendre comment générer et vérifier des hashes.

Résumé

Cette leçon vous a présenté les bases de la cryptographie et du hashing. Vous devez maintenant être capable de :

  • Comprendre la différence entre hash et chiffrement.
  • Hasher un mot de passe de manière sécurisée en JavaScript.
  • Choisir les bons algorithmes pour protéger les données sensibles.
